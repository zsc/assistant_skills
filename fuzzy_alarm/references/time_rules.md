# 模糊时间与日历规则（按需参考）

## 1) 工作日 / 平时 / 休息 / 周末

按优先级解释：

1. 用户自定义偏好（最高优先级）
2. 地区节假日/调休日历（若可用）
3. 默认回退
   - 工作日 = 周一到周五
   - 休息/周末 = 周六、周日
   - 平时 = 首次按“工作日”处理

对“平时”的处理：

- 首次未建模时，默认等同“工作日”，并在读回里明确：`我先按工作日处理。`
- 若用户纠正一次或多次，把“平时”映射为个性化日集合并沉淀为偏好。

## 2) 今天 / 明天 / 后天 / 下周

- 禁止创建到过去时间。
- “今天 + 模糊时段”但该时段已过去时：
  - **提醒**：顺延到最近可行未来时点，并明确说出顺延结果。
  - **闹钟**：不要静默顺延到明天；必须读回说明并给用户纠偏入口。
- “下周一/下个工作日/最近一个休息日”等必须落到未来日期。

## 3) 相对模糊时间（默认值，可被个人偏好覆盖）

- “待会儿 / 一会儿” → `now + 30 min`
- “过会儿” → `now + 20 min`
- “稍后” → `now + 15 min`
- “晚点” → 优先选 `+60~120 min` 内最近的低负荷可行点

规则：

- 若存在同类历史偏好，用偏好覆盖默认值。
- 驾驶高负荷时不展开追问：普通事项可直接用默认值并读回。
- 高重要性事项即使有默认值，也优先走显式确认/草稿策略（见 `references/importance_confirmation.md`）。

## 4) 模糊时段映射（窗口 + 默认锚点）

用户只说时段、不说精确时刻时，按以下窗口与锚点落地（可被偏好覆盖）：

| 表达 | 时间窗口 | 默认锚点 |
| --- | --- | --- |
| 凌晨 | 00:00-05:59 | 01:00 |
| 清晨 | 05:30-07:00 | 06:30 |
| 早上 | 07:00-09:30 | 08:00 |
| 上午 | 09:00-11:30 | 09:30 |
| 中午 | 11:30-13:30 | 12:15 |
| 下午 | 13:30-18:00 | 15:00 |
| 傍晚 | 17:30-19:30 | 18:00 |
| 晚上 | 19:00-22:00 | 20:00 |
| 夜里 | 22:00-24:00 | 22:30 |

## 5) “稍早不稍晚”（提醒） vs 保守锚点（闹钟）

- 对**提醒**：在窗口内可略偏前，推荐提前 `10-20 min`，但不要提前到违背常识。
- 对**闹钟**：不要因为“怕漏掉”擅自大幅前移；只有用户明确表达“早点/宁可早一点”才前移。

## 6) 个性化覆盖

用户明确声明时，优先使用并沉淀：

- “我说早上一般就是七点半” → `早上.anchor = 07:30`
- “中午对我来说是一点左右” → `中午.anchor = 13:00`
- “休息日别太早提醒我” → 调整休息日窗口内默认锚点
